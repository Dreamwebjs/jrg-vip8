2.如何去除列表前面的点或者数字？

使用 list-style:none

3.class 和 id 有什么区别？什么时候用 class 什么时候用 id？

区别：
1. 用于给一些元素加上共同的样式或者特性时，使用 class 选择器，一个 class 可以有多个元素，同样一个元素可以有多个 class 选择器
2. 而 id 则是一个独一无二的名字，用于给一个特殊的样式，并且 id 的权重高于 class

4.块级元素、行内元素是什么？有什么区别？分别对应哪些常用标签？

块级元素：独占一行，不管内容多少
行内元素：内部的元素依次排列，注意行内元素的高度是由其内容撑开的

  • 块级元素：div/p/h1~h6/table/tr/ul/li/dl/dt/form
  • 行内元素：a,span,img,input,button,em,textarea

5.display: block、display: inline、display: inline-block分别有什么作用?

display: block：表示把页面内容以块级元素显示
display: inline：表示把页面内容以行内元素显示
display: inline-block：让元素既有行内元素的紧跟其后的特性，又有块级元素的宽高margin，padding设置特性。

8.form表单有什么作用？有哪些常用的input 标签，分别有什么作用？

form作用：用于为用户输入创建 HTML 表单。表单用于向服务器传输数据。

input常用标签：
<input type="text" name="">：普通文本框
<input type="password" name="">：密码框
<input type="file" name="">：上传文件
<input type="button" name="">：普通按钮
<input type="radio" name="">：单选框
<input type="checkbox" name="">：复选框

input一定要有 name 属性，可以添加 disabled 来禁止用户输入

9.post 和 get 方式的区别？

两者都是将表单数据提交给服务器，但是有一些区别：
1. get 提交的数据追加到 URL 的最后，然后对服务器请求，而 post 则是将数据作为请求的一部分发送给服务器，用户是不可见的
2. get 一般是提交少量数据，而post 则是提交大量数据
3. get最多提交1K 的数据，受到浏览器的限制；而 post 理论上无限制，收到服务器的限制
4. get 提交的数据会被记录在浏览器的历史记录中，安全性不佳
5. 如果期望用户对提交表单后的页面加书签，则必须用 get，如果用 post，服务器返回的页面则无法被用户添加书签

10.在input里，name 有什么作用？

相当于表单和处理表单的服务器之间的一个粘合剂，提交表单的时候，浏览器会使用这些唯一的名字打包所有数据。浏览器会得到各部分数据，并用唯一的name属性作为这些数据的标签，然后把这些名字和值发送给服务器。

11.提交的三种有什么区别？

  • <button>提交</button>双标签内可以放置其他内容，如图片。
  • <a class="btn" href="#">提交</a>这个通过样式来控制变成按钮样式，得用js来实现数据交互。
  • <input type="submit" value="提交">放在form表单内，可以打包form表单内的数据
